# Product

## Register

product

## Users

JavaScript and TypeScript developers building AI agents, research systems, RAG pipelines, documentation indexes, QA tools, and controlled browser workflows. They need to find the right crawler surface quickly, copy a working example, understand the returned evidence, and verify the authority boundary before using it in production.

## Product Purpose

Cockroach Crawler gives AI applications a single Node.js toolkit for crawling, rendering, mapping, extracting, and preserving web evidence. The documentation succeeds when a developer can move from an explicit task to a runnable public API, understand the output and failure behavior, and discover adjacent capabilities without searching the repository.

## Brand Personality

Precise, technical, confident. The interface should feel like a dependable engineering manual: dense enough for experts, legible enough for first-time users, and explicit about what each surface does.

## Anti-references

- Documentation that hides primary routes inside collapsed accordion navigation.
- Giant single-page capability walls that force users to scan unrelated material.
- Generic SaaS card grids, decorative dashboards, and marketing copy that obscures the API.
- Visual-only hierarchy that does not map to semantic headings or keyboard navigation.
- Tiny, low-contrast technical text and horizontally overflowing code examples.

## Design Principles

- Show the documentation map. Main groups and their subpages remain visible on desktop.
- Name the task first. Page titles use the developer's job, not an abstract slogan.
- Keep proof beside instruction. Examples, outputs, failures, and authority boundaries stay close to the capability they describe.
- Make deep links dependable. Current location, headings, breadcrumbs, and table-of-contents links agree.
- Preserve the terminal identity without sacrificing long-form readability.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Maintain semantic landmarks and heading order, keyboard-visible focus, 44-pixel touch targets, text and control contrast, browser zoom and reflow, reduced-motion compatibility, accessible code regions, and usable navigation at 320-pixel mobile widths.
